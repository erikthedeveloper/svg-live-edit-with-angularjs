SVG - What is It?

According to MDN

Scalable Vector Graphics (SVG) is an XML markup language for describing two-dimensional vector graphics. SVG is essentially to graphics what XHTML is to text. SVG is similar in scope to Adobe's proprietary Flash technology, but what distinguishes SVG from Flash is that it is a W3C recommendation (i.e., a standard) and that it is XML-based as opposed to a closed binary format. It is explicitly designed to work with other W3C standards such as CSS, DOM and SMIL.

Caveats and Things to Remember About SVG vs. HTML

Syntax

Since SVG has its own comprehensive set of elements and attributes the syntax, at first, can be overwhelming.

Visit the Mozilla Developer's Network for comprehensive (head spinningly comprehensive) lists of SVG Elements and SVG Attributes

Coordinate System

Starting with the top left corner as (0,0) (x,y), the x value increases to the right and y value increases towards the bottom.

*Example: the bottom left of a square of 200px would be (0,200) and the top right would be (200,0)

Stacking and Layering

One important thing to remember is in what order the elements are "stacked". This can be confusing since the elements "on top" of your code will actually be on the bottom of the "stack" of your SVG elements.

The globally valid rule for SVG files is, that later elements are rendered atop previous elements. The further down an element is the more will be visible.

Rectangles

Note that only the width and height are required, yet there are many attributes available for declaration.

*Inspect the elements to better visualize the layering/rendering/stacking order


<svg width="600"  height="300">
  <rect width="600" height="100" fill="green" />
  <rect width="300" height="300" fill="blue" />
  <rect width="100" height="100" fill="red" stroke="black" x="60%" y="50%" rx="10" ry="10"></rect>
</svg>
